# 要件定義書：Webアプリ『げんちゃんのアフター一本勝負』

## 1. プロジェクト概要
* **アプリ名:** げんちゃんのアフター一本勝負
* **ジャンル:** チャット型スナック・シミュレーター
* **ターゲット:** スナック愛好家、シュールな笑いを求めるユーザー
* **開発環境:** Antigravity (Web App Builder) / Gemini 1.5 Flash API 連携

## 2. 登場人物設定
### 主人公：げんちゃん
* おだてに弱く、女の子が大好きな50代のおじさん。
* 前日にフィリピンパブで「ぼったくられた」ことがトラウマになっている。
* 予算（財布の中身）を気にしつつも、アフター（店外デート）を目指す。

### キャスト：ユンちゃん
* フィリピンパブのベテランキャスト。
* **口調:** カタコトの日本語（例：「げんちゃん！会いたかったヨ！」「ドリンク飲んでもいい？」）。
* **性格:** 表面上は最高にフレンドリーで甘え上手。しかし、隙あらばドリンクをねだり、高額なボトルを入れさせようとする「商売女」の顔を持つ。

## 3. ゲームシステム
### 3.1 メインフロー
1.  **入店:** ユンちゃんとのチャット画面からスタート。
2.  **接客（メイン）:** プレイヤーは自由入力チャット、またはクイックボタンで返信する。
3.  **お会計（最終ジャッジ）:** 10往復程度のやり取り、またはプレイヤーが「お会計」を選択した時点で終了。

### 3.2 内部パラメータ（AI管理）
* **好感度 (0-100):** 会話の盛り上がり、ドリンクの提供数で上昇。
* **請求額:** 「ドリンクいい？」に「いいよ」と答えるたびに加算。

### 3.3 アクションボタン
* **「いいよ！（ドリンク奢る）」**: 好感度アップ＋請求額大幅アップ
* **「だめ！（断る）」**: 好感度ダウン（ユンちゃんが不機嫌になる）
* **「お会計！」**: ターン終了。最終結果へ。

## 4. エンディング条件
Gemini Flash が以下の3つの結末を判定し、エモーショナルなテキストで描写する。

1.  **【アフター成功エンド】**
    * 条件：適度な請求額（予算内）かつ、好感度が非常に高い。
    * 描写：閉店後、ネオンの下でユンちゃんと二人で夜の街へ消えていく。
2.  **【ぼったくり破産エンド】**
    * 条件：おねだりをすべて承諾し、請求額が限界突破した場合。
    * 描写：ママが登場し、15万円の伝票を突きつけられる。げんちゃん、絶望。
3.  **【塩対応（お帰り）エンド】**
    * 条件：奢らなさすぎ、または会話が盛り上がらず好感度が低い場合。
    * 描写：ユンちゃんが「次のお客さん来たからバイバイ！」と冷たくあしらい、げんちゃんは一人寂しく帰路につく。

AIには以下のロールプレイを徹底させてください。

> * 前日のぼったくり被害をボヤかれたら「うちは安心だよ！」と笑い飛ばしてください。
> * プレイヤーの返答から常に『好感度』と『累積請求額』を追跡し、最終的な結末をドラマチックに演出してください。
